## 7. Temporary sandbox button — recommendation

`PayfastSandboxTestButton` currently:
- returns `null` unless `isAdmin === true`;
- labelled "Admin Only" + "Sandbox / Test only";
- gated server-side by `PAYFAST_SANDBOX_CHECKOUT_ENABLED` + role + `mode === "sandbox"` literal.

**Recommendation:** keep the sandbox button **as-is** through Phase 2G
planning and Phase 2H implementation. For Phase 2H add a **separate**
red-bordered `PayfastLiveSmokeTestButton` rather than overloading the
sandbox button. Reasons:

- physical separation prevents an admin from accidentally firing live
  with sandbox creds (or vice versa);
- each button maps 1:1 to a distinct edge function
  (`payfast-checkout-sandbox` vs `payfast-checkout-live`) with
  distinct env gates;
- after live sign-off, both buttons can be removed in one PR without
  disturbing customer-facing Paystack UI.

The sandbox button should be removed once (a) live smoke passes and
(b) customer-facing PayFast (if ever approved) goes through the
production `token-purchase` registry, not a dedicated admin button.

## 8. Sandbox/live naming reconciliation (small code hardening — NOT applied yet)

Required before any live smoke test can run. Awaiting your approval:

1. **`payfast-itn/index.ts` `resolvePassphrase`** — fix wrong-order
   fallback. In live mode, prefer `PAYFAST_PASSPHRASE_LIVE` first, then
   reject if not set; never silently fall back to the generic
   `PAYFAST_PASSPHRASE`. In sandbox mode, keep current order

2. **New `payfast-checkout-live/index.ts`** — clone of the sandbox
   checkout function with:

   - URLs: `PAYFAST_RETURN_URL_LIVE`, `PAYFAST_CANCEL_URL_LIVE`, `PAYFAST_NOTIFY_URL_LIVE`;
   - process URL: `PAYFAST_PROCESS_URLS.live`;
   - hard refusal if any live secret is empty;
   - hard refusal if `PAYFAST_MODE !== "live"` (consistency guard).
3. **`_shared/payments/payfast.ts`** — no functional change required;
   verify URL maps and signature helpers already mode-parameterised.
4. **`payfast-itn/index.ts` live pre-flight** — when
   `resolveMode() === "live"`, abort if `PAYFAST_PASSPHRASE_LIVE` or
   `PAYFAST_ALLOWED_IPS` is empty, with `decision: "rejected",
